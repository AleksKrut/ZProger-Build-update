Установка Proger Build (bspwm) на Arch Linux
Без шифрования диска (простая схема UEFI/GPT)

Оригинальное видео доступно по ссылке YouTube,
репозиторий с конфигурацией — GitHub.

Перед началом установки Proger Build необходимо установить несколько пакетов и редактор micro.

Если у вас есть второй компьютер или ноутбук, вы можете подключиться к серверу по SSH и просто вставлять нужные команды — это делается легко.

💻 Подготовка к подключению по SSH

Для удобного SSH-подключения рекомендуется использовать MobaXterm.

  1. Скачайте и установите MobaXterm с официального сайта.
  2. На сервере также потребуется установить плагин (или пакет) для корректной работы. Код для установки плагина на сервере:
pacman -Sy --noconfirm openssh

После нужно узнать какой у вас IP. Для этого введите команду:

ip address

Вам необходима строчка enp4s0 — там будет прописан ваш серверный IP.

🔌 Как будут выполнены все действия по установке

Открываем MobaXterm и переходим во вкладку Session, выбираем пункт SSH и заполняем:

  • Remote host* — ВАШ IP
  • Username — root
  • Port — 22

По окончании нажимаете ОК. Если всё правильно, в терминале потребуется ввести пароль от root. Задайте его на сервере командой:

passwd

Ввести нужно два раза. Не пугайтесь, если на экране ничего не появляется — в Linux не показывается, как вводится пароль.

📦 Пакеты и редактор, которые требуется установить до начала работы с сервером

Установка micro

Вставлять код в консоль сочетанием клавиш Shift+Ins

pacman -Sy micro

Для выхода из редактора micro используйте CTRL+Q.

Установка пакета dhcpcd

pacman -S dhcpcd

Установка пакета iwd

pacman -S iwd

Для чистки консоли используется команда clear.

💽 Разметка диска под UEFI GPT (без шифрования)

Важно: эта инструкция предполагает установку без шифрования диска. Используется классическая схема: раздел EFI (FAT32) и корневой раздел (ext4).

Если вы используете SSD, ваши разделы будут выглядеть примерно так:

  • /dev/nvme0n1p1 — EFI
  • /dev/nvme0n1p2 — корневой (root)

В таком случае замените /dev/sda на /dev/nvme0n1, а разделы — на соответствующие.

Для просмотра дисков введите:

lsblk

У вас появится что-то вроде:

NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
loop0    7:0    0 955.7M  1 loop /run/archiso/airootfs
sda      8:0    0 465.8G  0 disk
└─sda1   8:1    0 465.8G  0 part
sdb      8:16   1   7.6G  0 disk
└─sdb1   8:17   1   7.6G  0 part

Выбираете название диска, который хотите использовать. В примере это sda.

Приступаем к разметке:

parted /dev/sda
mklabel gpt
# пропишите Yes
mkpart ESP fat32 1MiB 512MiB
set 1 boot on
mkpart primary ext4 512MiB 100%
quit

В результате у вас будут:

  • /dev/sda1 — раздел EFI (будет смонтирован в /boot)
  • /dev/sda2 — корневой раздел (ext4)

📂 Форматирование и монтирование разделов

Форматируем корневой раздел в ext4, а EFI — в FAT32:

mkfs.ext4 /dev/sda2
mkfs.fat -F 32 /dev/sda1

Монтируем разделы для установки системы:

mount /dev/sda2 /mnt
mkdir /mnt/boot
mount /dev/sda1 /mnt/boot

⚙️ Сборка ядра и базовых софтов

pacstrap -K /mnt base linux linux-firmware base-devel dhcpcd net-tools iproute2 networkmanager vim micro efibootmgr iwd
genfstab -U /mnt >> /mnt/etc/fstab
cat /mnt/etc/fstab
arch-chroot /mnt

Нужно раскомментировать ru_RU и en_US в формате UTF-8 в файле /etc/locale.gen. Для поиска используйте Ctrl+F:

micro /etc/locale.gen

Генерируем локали:

locale-gen

Настраиваем время:

ln -sf /usr/share/zoneinfo/Europe/Kiev /etc/localtime
hwclock --systohc

Указать имя хоста:

echo "arch" > /etc/hostname

Укажите пароль для root (этот шаг мы уже делали в самом начале):

passwd

Добавляем нового пользователя и настраиваем права:

useradd -m -G wheel,users,video -s /bin/bash user
passwd user
systemctl enable dhcpcd
systemctl enable iwd.service

Редактируем /etc/mkinitcpio.conf — для системы без шифрования:

micro /etc/mkinitcpio.conf

Найдите строку HOOKS=(base udev autodetect modconf kms keyboard keymap consolefont block filesystems fsck)
Оставьте её как есть, не добавляя encrypt и lvm2. Если они были, удалите.

Запустить пересборку ядра:

mkinitcpio -p linux

🖥️ Установка загрузчика

bootctl install --path=/boot
cd /boot/loader
micro loader.conf

Вставляем в loader.conf:

timeout 3
default arch

Создаём конфигурацию для запуска:

cd /boot/loader/entries
micro arch.conf

Вставляем в arch.conf следующее (UUID можно узнать командой blkid /dev/sda2):

title Arch Linux by ZProger
linux /vmlinuz-linux
initrd /initramfs-linux.img
options root=UUID=uuid_от_/dev/sda2 rw

Сохраняем и выходим. После вводим:

blkid /dev/sda2 >> arch.conf

Заходим обратно в arch.conf и заменяем uuid_от_/dev/sda2 на полученный UUID (например, UUID=1234-5678-...).

Выдаем права на sudo:

sudo EDITOR=micro visudo

После открытия раскомментируйте %wheel ALL=(ALL:ALL) ALL.

Выходим из системы и перезагружаемся:

Ctrl+D   # или exit
umount -R /mnt
reboot

🔄 После перезагрузки для повторного подключения по SSH введите следующие команды

sudo systemctl enable sshd    # включить автозапуск при загрузке
sudo systemctl start sshd     # запустить службу сейчас

🎨 Устанавливаем оболочку

При загрузке системы войдите под пользователем user (пароль, который задавали). Выполните:

sudo pacman -Syu
sudo pacman -S xorg bspwm sxhkd xorg-xinit xterm git python3

Настройка xinitrc:

micro /etc/X11/xinit/xinitrc

Отключите любые другие строки exec и добавьте в конец файла:

exec bspwm

Загрузите репозиторий локально. Перед выполнением билдера рекомендую заглянуть в Builder/packages.py и посмотреть устанавливаемые пакеты. BASE_PACKAGES лучше не редактировать, остальные — на ваше усмотрение.

git clone https://github.com/Zproger/bspwm-dotfiles.git
cd bspwm-dotfiles
python3 Builder/install.py

В меню предоставьте разрешение на установку dotfiles, обновление баз, установку BASE_PACKAGES. Остальные пункты выбирайте сами.

Если всё сделано правильно, после запуска получите готовую оболочку BSPWM:

startx

Из-за разного железа могут быть небольшие проблемы с отображением иконок, батареей/яркостью. Решение показано в данном видео.

Готовая система без шифрования с bspwm от ZProger. Удачи в настройке!
